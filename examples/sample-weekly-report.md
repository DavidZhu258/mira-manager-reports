# Weekly Project Report

## One-line Summary
This week packaged daily engineering activity into a reusable reporting workflow with GPT-token summaries, local evidence files, and scheduled Windows automation.

## Main Outcomes
- Built a weekly report workflow for multi-root project scanning.
- Added scheduled daily and weekly Windows Task Scheduler helpers.
- Published examples that show both the manager report and the raw activity trail.

## Important Technical Changes
- `ops/run-project-report.ps1` gathers file activity and optional profile context.
- `ops/run-scheduled-project-report.ps1` creates the output folder and desktop shortcut.
- `ops/install-report-scheduled-tasks.ps1` registers daily and weekly scheduled tasks.

## Product / User Impact
- A team lead can review weekly progress without opening each repository.
- Reports remain portable because the raw JSON evidence is saved beside Markdown.

## Quality, Verification, and Deployment
- Tests cover scheduled config and file-activity generation.
- The direct API path redacts bearer tokens from error messages.

## Risks / Blockers
- Model output quality depends on the supplied activity JSON and prompt.
- Private paths should be avoided before committing generated reports.

## Next Week Focus
- Add optional CI examples and richer report templates.
